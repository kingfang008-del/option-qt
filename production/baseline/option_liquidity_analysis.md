# 期权流动性分析总结（基于当前基线策略）

## 1. 分析目标

本报告用于沉淀当前策略中关于“期权流动性”的核心结论，重点回答三个问题：

1. 什么时候我们会认为期权“可交易”？
2. 资金规模变大后，订单是否会被动拆单，容量上限在哪里？
3. 历史回测中，流动性对收益和回撤的实际影响是什么？

---

## 2. 当前流动性判定口径

### 2.1 开平仓点差门槛

- 开仓：`MAX_SPREAD_PCT_ENTRY = 0.10`（约 10%）
- 平仓：`MAX_SPREAD_PCT_EXIT = 0.20`（约 20%，放宽以保证风险退出）
- 点差异常发散：`MAX_SPREAD_DIVERGENCE = 0.02`

### 2.2 低价期权过滤

- `MIN_OPTION_PRICE = 1.0`
- 低于该价格的合约会被拒单，避免“跳动粗 + 点差占比高”的流动性陷阱。

### 2.3 订单容量与拆单机制（LiquidityRiskManager）

订单通过 `evaluate_order()` 后，不会直接砍掉总金额，而是通过 `chunks` 进行冰山拆单建议。

- 若有盘口深度（`ask_size > 0`）：按卖一挂单手数动态计算单笔安全容量。
- 若无深度（回测/缺失 L2）：走静态 Tier 兜底。

静态 Tier 免拆上限：

- Tier1（`SPY/QQQ/NVDA/TSLA/AAPL/IWM`）：`$30,000`
- Tier2（`META/AMZN/MSFT/AMD/MSTR/COIN/SMCI`）：`$10,000`
- Tier3（其他）：`$3,000`

---

## 3. 回测侧流动性影响结论

### 3.1 容量检查指标（系统已有）

回测报告中，流动性影响模块主要关注：

- `Splitted Trades (Iceberg)`：触发拆单的交易数量与占比
- `Average Entry Capital`：平均单笔开仓名义资金
- `Avg Capital triggering Split`：触发拆单时的平均名义资金
- `Max Chunks Observed`：最大拆分笔数

口径上，若拆单占比高于 30%，系统会提示容量预警（建议降仓位或集中高流动性标的）。

### 3.2 历史风险样本（交易日志）

- 2026-03-12 出现 `UNH Liquidity Spike`，当日记录约 `-6.6%`，单笔示例 `UNH call` 约 `-20.6%`。
- 该类样本说明：非核心高流动性标的在局部时段可能出现点差突然拉宽或盘口承接不足，导致退出质量显著恶化。

### 3.3 与市场状态过滤的联动

Regime Guard 回测对比（2026-01 与 2026-03 的 13 个交易日）显示：

- 在高噪音环境下可明显减少无效交易与回撤（总交易数下降约 26%）。
- 本质上也间接降低了“在差流动性/高噪声时段强行成交”的概率。

---

## 4. 可执行建议（按优先级）

1. 优先控制“标的流动性”，再放大仓位  
   大资金先集中 Tier1/Tier2，减少 Tier3 曝光。

2. 若拆单占比持续偏高，先降 `POSITION_RATIO`，再考虑扩容  
   比直接提高 `MAX_TRADE_CAP` 更稳健。

3. 维持较严开仓点差阈值（`MAX_SPREAD_PCT_ENTRY`）  
   它对抑制坏成交最有效，通常比事后止损更便宜。

4. 对“流动性异常日”单独做复盘标签  
   建议持续记录：点差分位数、ask_size 缺失率、拆单占比、退出滑点。

---

## 5. 一句话结论

当前策略的流动性框架已经具备“三层保护”：`价格底线 + 点差过滤 + 容量拆单`。下一阶段优化重点不在“更激进下单”，而在“减少低质量时段交易 + 提高标的池流动性密度”。
